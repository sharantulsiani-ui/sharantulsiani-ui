## How I work with AI builders, the short version

1. I only build things I personally need every day. Success means "I use it daily", not "it runs".
2. Constraints get written down before architecture. Privacy boundaries, my attention limits, what always needs my approval.
3. The AI works under a written contract: a control file with one active milestone at a time, clear pass criteria, and a standing rule that the file outranks anything said in chat.
4. Nothing unproven touches the live system. New capability runs in parallel until it passes a gate on real cases, judged by me.
5. Every mistake becomes a written rule, with the incident that caused it attached.

The honest version of what I do and don't do: the AI writes the code and most first drafts. I write the decisions, the constraints, the gates and the rejections. I keep those separable on purpose.
